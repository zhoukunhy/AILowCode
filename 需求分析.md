## 1 引言

### 1.1 项目背景

市面传统低代码平台偏重拖拽、AI能力薄弱，无法根据自然语言自主完成建表、生成接口、生成前端页面、代码导出、故障自查；普通开发者从零开发CRUD页面重复工作量大。本项目主打**Agent自主驱动+可视化拖拽**双模式，依托LangGraph多智能体、RAG私有知识库、AST代码生成，打造一款开源全栈AI低代码画布，可作为求职全栈标杆开源项目。

### 1.2 文档目的

约束开发范围、明确前后端+AI各模块需求，作为Trae分阶段开发提示词依据、自测与验收标准。

### 1.3 术语定义

| 术语 | 释义 |
|------|------|
| CanvasSchema | 画布页面结构化JSON，存储组件位置、属性、数据源绑定配置 |
| RAG | 私有知识库：文档切片→向量化→Milvus存储→语义召回 |
| LangGraph Agent | 基于状态机的多节点智能体，流程可分支、循环、自检重试 |
| AST代码生成 | 解析画布JSON，自动生成React/Nest源码与SQL建表语句 |
| Monorepo | pnpm+Turbo分包管理，apps业务应用+packages公共工具包 |

## 2 项目总体概述

### 2.1 项目目标

1. **可视化画布**：拖拽式搭建Web页面，实时预览、属性配置、页面版本管理；
2. **AI全链路赋能**：自然语言→LangGraph Agent自动生成画布、建数据库表、生成后端接口、绑定数据源；
3. **RAG私有知识库**：上传项目规范/API文档/需求文档，AI生成内容参考内部知识库，减少幻觉；
4. **一键源码导出**：基于AST自动生成可独立运行React前端+Nest后端工程；
5. **智能故障诊断Agent**：数据源报错/接口异常自动检索知识库，给出修复方案；
6. **开源可私有化部署**，Docker一键启动全套服务。

### 2.2 运行环境

- **开发**：Node.js≥20、pnpm、Docker/Docker-Compose
- **数据库**：PostgreSQL、Redis、Milvus向量数据库
- **部署**：Linux/Windows、Docker容器、支持Render免费云端部署演示站点

## 4 整体架构与技术约束

### 4.1 整体架构（Monorepo）

```
canvas-code/
├─apps/
│  ├─canvas-editor(Next15+React18画布编辑器)
│  ├─admin-dashboard(AntDesign Pro管理后台)
│  └─nest-server(NestJS后端主服务)
├─packages/
│  ├─lang-ai-core【核心：LangChain+LangGraph+RAG+Agent】
│  ├─ai-sdk【LLM底层封装，多模型统一调用】
│  ├─code-generator【AST代码生成引擎】
│  ├─shared-types【全栈统一TS类型】
│  ├─ui-components【公共组件库】
│  ├─datasource-core【数据源连接引擎】
│  └─common-utils【通用工具】
```

### 4.2 强制技术约束

1. 全项目TypeScript，禁止JS裸写；
2. 前端：Next15(App Router)+Zustand+Tailwind+Konva画布；
3. 后端：NestJS+TypeORM；缓存Redis；
4. AI层：langchain、@langchain/langgraph、Milvus向量存储；
5. 容器：docker-compose一键拉起PG/Redis/Milvus全环境；
6. 接口统一返回格式：`{code:number,msg:string,data:any}`，全局异常拦截。

## 7 非功能性需求

### 7.1 性能需求

| 指标 | 要求 |
|------|------|
| 画布常规操作响应 | ≤300ms |
| AI生成页面等待 | ≤5s |
| RAG向量检索单文档召回 | ≤800ms |
| 单画布组件数 | 支持50+组件流畅渲染，大数据列表懒加载优化 |

### 7.2 安全性

| 需求 | 说明 |
|------|------|
| 用户密码 | 加密存储 |
| 接口鉴权 | JWT鉴权、RBAC接口权限拦截 |
| LLM密钥 | 后台加密存储，前端不可查看 |
| 数据源密码 | 加密入库，防明文泄露 |

### 7.3 可靠性

| 需求 | 说明 |
|------|------|
| 降级机制 | Milvus/PG异常降级提示，AI服务不可用时切换备用LLM |
| 自动草稿保存 | 画布异常断电不丢失配置 |
| 死循环防护 | LangGraph节点异常捕获、循环次数限制 |

### 7.4 可扩展性

| 需求 | 说明 |
|------|------|
| LLM工厂模式 | 新增模型仅需在ai-sdk新增适配器 |
| LangGraph可扩展 | 可新增自定义Agent节点，无需改动主流程 |
| Wasm插件市场 | 可后续拓展自定义组件插件市场 |

### 7.5 易用性

| 需求 | 说明 |
|------|------|
| 全中文提示 | 前端全中文提示 |
| AI助手侧边栏 | 实时对话修改页面 |
